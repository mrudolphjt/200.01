You are here to access eSeries securely.

Before you can work effectively in eSeries, you need to understand how the platform handles login, security configuration, account recovery, and logout.

Authentication is not just a sign-in step. It is part of protecting court data, user access, and the integrity of each work session.
👤

Execute first-time login and complete initial security configuration.

🔐

Manage internal and external login methods across the platform.

Security

Execute formal logout and manage session persistence correctly.

Start with the full authentication lifecycle.

Onboarding

Your first login establishes your security profile.

During onboarding, your program manager provides your username and temporary password. Keep these credentials secure until your first login.

Your temporary password is single-use. You will be required to set a permanent password immediately upon first login.

1
Open the welcome link.
Use the welcome link provided during onboarding.
2
Create your password.
Choose a strong, unique password that meets platform requirements.
3
Configure Secret Questions.
These enable future self-service password resets.
Authentication

Read the login splash screen before you authenticate.

The login splash screen supports internal credentials, external identity providers, and environment diagnostics for troubleshooting.

Internal Auth

Use standard username and password credentials stored within eSeries.

External Auth

Use supported providers such as Azure AD, LDAP, and Active Directory.

Diagnostics

Environment version and build numbers appear at the bottom for troubleshooting.

Look near the bottom of the login splash screen. Capture the version and build number when troubleshooting or reporting environment-specific behavior.
Security policy

“Stay Logged In” changes how long the session persists.

The “Stay Logged In” option uses browser local storage to maintain the session even after the browser is closed.

Closing the browser is not the same as logging out.
!
Corporate policy
Never use “Stay Logged In” on public or shared hardware. Always perform a manual logout at the end of every session.

Knowledge check

You are using a shared testing machine. Should you select “Stay Logged In”?

External authentication

External login uses your corporate identity provider.

1
Enter your corporate email.
Use your corporate email as the username.
2
Leave the password field blank.
External authentication does not use the internal password field.
3
Select the provider.
Click the specific provider button, such as “Log In With Azure.”
i
Administrator setup required
Your account must be pre-configured by an administrator before external authentication can be used.
Maintenance

Maintain your access before there is a problem.

eSeries supports self-service password recovery and internal profile security updates. Keep your Secret Questions current so account recovery works when you need it.

1

Initiate

Click “I forgot my password” on the login splash screen.

2

Verify

Answer your pre-configured Secret Questions to verify your identity.

3

Recover

Use the reset link sent to your registered email address.

✓
Save required
Profile security changes are only applied after clicking “Save & Close.” Closing without saving discards all updates.
Formal logout

End every session with a manual logout.

Manual logout ensures the session token is destroyed immediately, reducing the risk of session hijacking on shared hardware.
1
Execute
Locate the Logout icon in the Top Navigation bar of the dashboard.
2
Confirm
Confirm the action in the pop-up prompt to invalidate your session token.
3
Secure
Always perform a manual logout, especially on shared or public hardware.
